# 项目报告：手机端雾霾天气探测系统

## 一、项目概述

本项目为一个适配手机端的雾霾天气探测系统，使用 Node.js + Express 构建后端服务，前端使用原生 HTML5/CSS3/JavaScript 实现移动端页面。系统可获取定位城市、保存城市信息、查询天气与空气质量数据，并以手机友好的方式展示。

## 二、系统功能

1. 页面适配手机屏幕，使用响应式布局。
2. 浏览器 Geolocation API 获取当前经纬度。
3. 采用 OpenStreetMap 反向地理编码获取城市名。
4. 定位失败时提供手动输入城市功能。
5. 前端将定位城市发送到后端保存。
6. 后端提供如下接口：
   - `POST /api/location`：保存当前城市。
   - `GET /api/location`：获取保存的城市。
   - `GET /api/weather?city=城市名`：查询天气和空气质量。
   - `GET /api/data`：查看服务器端保存的完整数据。
7. 支持 mock 数据模式，保证没有天气 API Key 时也能正常演示。
8. `.env` 预留真实 API Key 配置：`WEATHER_API_KEY`、`WEATHER_PROVIDER`、`GEOCODE_API_KEY`。

## 三、技术实现

- 后端使用 `express` 提供 API 服务，`dotenv` 加载配置。
- 简易数据库使用 `data.json` 存储定位城市、天气和空气质量信息。
- 前端页面展示当前城市、日期、天气详情、空气质量、趋势图和出行建议。
- 采用 Canvas 绘制温度和湿度趋势折线图，增强移动端可视化效果。

## 四、运行方式

1. 安装依赖：`npm install`
2. 复制配置文件：`cp .env.example .env`
3. 调整 `.env`：
   - `USE_MOCK=true`：启用演示模式
   - `WEATHER_API_KEY`：接入真实天气 API 时填写
4. 启动服务：`npm start`
5. 浏览器访问：`http://localhost:3000`

## 五、演示说明

- 首次打开页面时，系统尝试自动定位并获取城市。
- 如果定位失败，则显示城市输入表单。
- 当前城市会显示在页面顶部，并保存到服务器端。
- 点击“刷新”按钮可重新拉取天气与空气质量数据。
- 页面会根据 AQI 等级显示健康与出行建议。

## 六、课程设计亮点

- 前后端分离设计，结构清晰。
- 使用纯前端技术实现移动端体验，便于教学展示。
- mock 数据模式保证演示稳定，无需依赖外部 API。
- 预留真实 API 接入方式，兼顾课程设计与实际扩展。
